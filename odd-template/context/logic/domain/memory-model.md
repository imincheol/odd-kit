# 🧠 The Memory Model (메모리 모델)

> **"기억하는 방식이 곧 존재의 방식이다."**
> ODD 시스템은 인간의 뇌 구조를 모방하여 설계되었습니다.

## 1. Structure (구조)

### 🧠 Context (The Brain - 맥락)
- **비유**: 지능의 근원, 장기 메모리
- **성격**: 가변성(Mutable), 현행화된 진실(Current Truth).
- **구성**: **History(역사)**, **Protocols(규약)**, **Logic(논리)**, **General(개요)**.
- **규칙**: 에이전트는 명령을 수행하기 전 반드시 관련 맥락을 정독해야 합니다.

### ⚡ Orders (The Action - 명령)
- **비유**: 행동 지침, 작업 기억
- **성격**: 목적 지향성, 현재 진행 중인 변화.
- **구성**: **Roadmap(지도)**, **Order(명령서)**.
- **규칙**: 승인되지 않은 오더는 실행될 수 없습니다.

### 📦 Archive (The Evidence - 증거)
- **비유**: 블랙박스, 원본 데이터
- **성격**: 불변성(Immutable), 객관적 증거.
- **구성**: 완료된 태스크 파일, 맥락의 개정판 스냅샷.
- **규칙**: 감찰관만이 이곳의 데이터를 분석하여 맥락을 정제합니다.

## 2. Information Flow (정보의 흐름)
데이터는 다음과 같이 순환하며 지혜가 됩니다.

1.  **Input**: 사용자의 요구사항이나 기획이 `Specs`에 등록됩니다.
2.  **Order**: `Context`를 참조하여 작업을 수행하기 위한 공식 명령(`Order`)이 생성됩니다.
3.  **Process**: 학생 페르소나가 명령을 수행하며 로그를 남깁니다.
4.  **Archive**: 작업 종료 후 모든 기록은 `Archive`로 이동하여 보존됩니다.
5.  **Refine (Audit)**: **감찰관(Auditor)**이 증거를 분석하여 새로운 지식을 `Context`에 반영합니다.
6.  **Next Action**: 갱신된 `Context`는 다음 오더의 상위 맥락이 되어 시스템의 지능을 높입니다.
