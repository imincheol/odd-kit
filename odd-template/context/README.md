# 🧠 ODD Context (v1.0.0)

> **"지능은 맥락 속에서만 발현됩니다."**

이곳은 프로젝트의 **활성화된 두뇌(Active Brain)**입니다. 단순히 문서를 쌓아두는 곳이 아니라, 에이전트가 오더(Order)를 수행할 때 반드시 참조해야 할 **살아있는 지식 집합체**입니다.

---

## 🏛️ Context Pillars (맥락의 기둥)

### 1. 🏛️ [General](./general/) (개요)
- **정체성**: 프로젝트의 비전, 팀 문화, 외부 환경 등 포괄적인 정체성.
- **역할**: "이 프로젝트가 왜 존재하는가?"에 대한 근본적인 배경을 제공합니다.

### 2. 📜 [History](./history/) (역사서)
- **과거의 흐름**: 프로젝트에서 발생한 주요 결정들의 연대기.
- **역할**: "우리는 왜 여기까지 왔는가?"에 대한 맥락적 해석입니다.

### 3. ⚙️ [Protocols](./protocols/) (규약)
- **행동 표준**: 에이전트와 사용자가 약속한 협업 방식과 운영 규칙. (Operation Logic)

### 4. 🏗️ [Logic](./logic/) (논리)
- **시스템의 진실**: 실제 구현된 비즈니스 논리와 기술 구조. (System Logic)
- **Relationship**: 기획서(Specs)가 요구사항이라면, 로직(Logic)은 그 요구사항이 수용된 후의 **현재 상태**입니다.
코드로 구현되는 과정에서 축적된 **현실적인 동작 원리**입니다.

---

## 🛡️ 감찰관(Auditor)의 행동 수칙

1. **맥락의 최신화**: 모든 업무가 완료되면 아카이브를 분석하여 변화된 내용을 즉시 **Logic** 서가에 반영하십시오.
2. **역사의 기록**: 중요한 설계의 변경이나 결정 사항은 **History** 서가에 연대기 순으로 기록하여 맥락이 끊기지 않게 하십시오.
3. **증거 보존**: 가공되지 않은 원본 오더와 리포트는 `archive/tasks/`로 이동시켜 증거로 남기십시오.
