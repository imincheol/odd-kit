# 📚 ODD Library (v1.1.0)

> **"지식은 단순한 축적이 아니라, 정제와 연결의 산물입니다."**

이곳은 프로젝트의 **장기 기억(Long-term Memory)**을 담당하는 도서관입니다. 
단순한 데이터 보관소가 아닌, 에이전트가 "현재의 상태"와 "과거의 맥락"을 이해하기 위한 모든 지식을 품고 있습니다.

---

## 📖 Bookshelf (서가 구조)

### 1. 📜 History (역사서)
- **Chronicles**: 발생한 주요 사건과 결정들의 연대기.
- **Interpretation**: 과거의 결정이 현재 시스템에 어떤 영향을 주었는지에 대한 해석.
- **Rule**: 과거의 사실은 변하지 않으나, 새로운 통찰을 담은 다른 관점의 역사서가 추가될 수 있습니다.

### 2. ⚙️ System (운영 체계)
- **Protocol**: ODD 운영 매뉴얼 및 에이전트(Persona)별 가이드.
- **Architecture**: 기억(ODD)과 진실(Specs)의 분리 설계 및 상호 참조 원칙.

### 3. 💼 Domain & 🛠️ Tech (실질 지식)
- **Living Knowledge**: 현재 프로젝트가 어떻게 동작하고 있는가? (Overwrite 방식의 최신화)
- **Link**: 기획서(`SPECS_DIR`)와 실제 구현 코드 사이의 간극을 메우는 지식.

---

## ✍️ 사서의 가이드 (Librarian Protocols)

1. **증거와 지식의 분리**: 업무 수행 중 발생한 파편적 로그와 오더 원본은 `archive/tasks/`로 보냅니다. 라이브러리에는 오직 정제된 **역사와 지식**만 남깁니다.
2. **최신성 수호**: `domain/`과 `tech/` 서가는 언제나 "지금 이 순간"의 프로젝트 정체성을 대변해야 합니다.
3. **역사의 기록**: 모든 오더의 완료 보고(`report`) 이후, 핵심 내용은 반드시 `history/`의 역사서에 연대기 순으로 기록되어야 합니다.
4. **시냅스 연결**: 책과 책 사이, 그리고 책과 기획서 사이의 하이퍼링크를 통해 지식의 연결망을 강화하십시오.
