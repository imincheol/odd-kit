# 🔍 ODD Reference (참조 정보)

> **"지능은 명확한 근거(Reference) 위에서 작동합니다."**

이곳은 프로젝트의 모든 **현재 지식**과 **기준**이 보관되는 곳입니다. AI 에이전트는 작업을 시작하기 전 이곳의 정보들을 참조하여 '잘못된 추측'을 방지해야 합니다.

---

## 📂 Reference Structure (참조 구조)

### 1. 📋 [Summaries](./summaries/) (요약 정보)
- **Status**: 현재 시스템이 어떻게 구현되어 있는지에 대한 압축된 정보.
- **Content**: 비즈니스 로직, 데이터 구조, API 명세 등 (As-is).

### 2. ⚖️ [Rules](./rules/) (규칙)
- **Standard**: 개발 시 지켜야 할 컨벤션 및 AI 에이전트의 행동 지침.
- **Content**: 코딩 스타일, 에러 처리 규칙, ODD 운영 프로토콜 등.

### 3. 📜 [Records](./records/) (기록물)
- **Decision**: 과거의 주요 기술적 의사결정 및 프로젝트의 흐름.
- **Content**: 결정 사항(ADR), 주요 마일스톤 히스토리.

---

## 👩‍🏫 AI 에이전트 지침

1.  **참조 우선**: 어떤 코드를 수정하기 전, 반드시 `summaries/`에서 현재 구현 방식을 확인하십시오.
2.  **규칙 준수**: 모든 결과물은 `rules/`에 정의된 가이드라인을 벗어나지 않아야 합니다.
3.  **지식 동기화**: `tasks/`에서 수행된 작업 결과는 즉시 `summaries/`에 반영하여 '정보의 최신성'을 유지하십시오.
