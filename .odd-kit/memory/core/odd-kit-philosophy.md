# 🧠 Core Philosophy: The 3-Pillar Temporal Hierarchy

## 1. 개요
ODD Kit은 시간을 기준으로 프로젝트의 지식을 세 가지 기둥(Pillar)으로 분류하여 관리합니다.

## 2. 세 가지 기둥 (3 Pillars)
1. **`docs/` (Present - 설계)**: 프로젝트의 현재 청사진. "스펙에 없는 것은 존재하지 않는다."
2. **`odd-kit-template/` (Future - 배포)**: 프로젝트의 결과물이자 타 프로젝트로 전파될 배포 원형. 리드미와 설계의 실제 구현 가이드.
3. **`.odd-kit/` (Past~Now - 엔진)**: 프로젝트를 실제로 빌드해온 과정(기록)과 이를 가동하는 규칙(엔진).

## 3. 리드미의 역할 (External Manual)
- **정체성**: 외부 사용자와 사용자(나)를 위한 가장 강력한 메뉴얼.
- **Bilingual Protocol**: 영어(세계인용)와 한국어(사용자용)를 1:1로 동일하게 유지. 이는 AI가 프롬프트를 통해 내용을 참조할 때도 강력한 일관성을 제공함.
- **Mirroring**: 리드미는 `docs/specs/` 전문 명세서의 요약본이자 거울이어야 함.

## 4. 엔진의 역할 (Active Logic)
- `.odd-kit/`은 일상의 작업이 일어나는 '작업 기억(Working Memory)' 공간임.
- 작업을 통해 얻은 경험은 리포트를 거쳐 원자적 **Memory Cell**로 응축됨.
