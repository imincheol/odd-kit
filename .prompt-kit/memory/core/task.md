# 🛠️ Task: 최소 작업 단위

## 1. 정의
Task는 더 이상 쪼개질 수 없는 논리적인 작업의 최소 단위입니다. Order는 여러 개의 Task로 구성될 수 있습니다.

## 2. 핵심 역할
- **실행 가능성**: 구체적인 명령어(Command)나 수정 대상이 명확해야 합니다.
- **병렬성 관리**: 작업 간의 의존성을 파악하여 순차적 혹은 병렬적 실행을 지원합니다.

## 3. 연관성
- **Active (0.90)**: 현재 수행 중인 태스크는 Active 디렉토리에서 관리됩니다.
- **Progress (0.85)**: 태스크의 수행 결과는 프로그레스에 실시간으로 반영됩니다.
