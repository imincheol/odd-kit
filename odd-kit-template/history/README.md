# 🗄️ History (기록 보관소)

> **"지나간 기록은 미래의 문제를 해결하는 열쇠입니다."**

이곳은 프로젝트에서 수행된 모든 작업의 **원본 증거(Evidence)**와 **이전 지식의 스냅샷**이 보관되는 공간입니다. 

---

## 📂 History Structure

### 1. [Tasks](./tasks/)
- 완료된 모든 Order, Progress, Report 파일이 날짜별(`YYYY/MM/DD`)로 저장됩니다.
- "우리가 과거에 어떤 과정을 거쳐 이 기능을 만들었는가?"에 대한 답을 제공합니다.

### 2. [Reference Revision](./reference/revision/)
- 요약 정보(`summaries`)나 규칙(`rules`)이 큰 폭으로 업데이트될 때, 이전 버전의 파일을 백업합니다.

---

## 👩‍🏫 관리 지침

1.  **불변성**: 히스토리에 한 번 들어온 파일은 수정하지 않는 것이 원칙입니다.
2.  **색인**: `PROMPT_KIT.md`나 `tasks/roadmap.md`를 통해 과거 기록을 쉽게 찾아볼 수 있도록 관리하십시오.
