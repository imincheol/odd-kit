# 📋 Summaries (요약 정보)

> **"복잡한 시스템의 현재 모습을 한눈에 파악합니다."**

이곳은 프로젝트의 개별 기능, 데이터 구조, 비즈니스 로직에 대한 **최신 요약본**들이 보관되는 곳입니다. 

---

## 📂 Summary Categories

### 1. [Domain](./domain/)
- 비즈니스 도메인 지식 및 핵심 서비스 로직 요약.

### 2. [Tech](./tech/)
- 기술 아키텍처, 사용 중인 라이브러리 활용법, 데이터베이스 스키마 요약.

---

## 👩‍🏫 업데이트 규칙 (Librarian's Rule)

1.  **압축**: 수천 줄의 코드를 설명하는 대신, 핵심 설계 철학과 동작 방식 위주로 압축하여 기록하십시오 (AI의 컨텍스트 절약 목적).
2.  **현행화**: `tasks/`에서 관련 기능 수정이 완료되면, 즉시 해당 요약 파일을 업데이트해야 합니다.
3.  **참조**: 새로운 오더를 작성할 때, 연관된 요약 파일의 경로를 `context` 섹션에 포함시키십시오.
