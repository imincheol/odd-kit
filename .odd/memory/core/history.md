# 🗄️ History: 과거의 축적

## 1. 정의
History는 완료된 모든 작업 데이터(Order, Report, Progress)가 아카이브되는 공간입니다.

## 2. 핵심 역할
- **추적성(Traceability)**: 과거에 어떤 의사결정과 코드가 변경되었는지 소급 추적을 가능케 합니다.
- **학습 데이터**: 에이전트가 과거의 실수나 성공 패턴을 학습하는 기반이 됩니다.

## 3. 연관성
- **Report (0.85)**: 리포트가 히스토리에 저장되는 핵심 문서입니다.
- **Reference (0.75)**: 히스토리의 데이터 중 가치 있는 것은 Reference로 정제됩니다.
