# 📋 Order: 실행의 기점

## 1. 정의
Order는 시스템에 내리는 구체적인 작업 지시서입니다. 사용자의 요구사항을 AI가 이해할 수 있는 명확한 Task 단위로 변환한 문서를 의미합니다.

## 2. 핵심 역할
- **목표 설정**: 작업의 최종 결과물(Target)을 명시합니다.
- **컨텍스트 명시**: 작업에 필요한 참고 파일 및 배경 정보를 연결합니다.
- **제약 조건**: 작업 시 준수해야 할 규칙을 포함합니다.

## 3. 연관성
- **Progress (0.95)**: 오더가 실행되면 진행 상황을 기록하는 Progress와 즉각적으로 연결됩니다.
- **Report (0.90)**: 오더의 완료는 Report 작성을 통해 증명됩니다.
