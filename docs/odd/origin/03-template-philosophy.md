# Origin: odd-kit-template Philosophy

> **Date**: 2026-01-04
> **Source**: User's Direct Instruction

당연히 odd-kit-template 이지 
.odd-kit 은 사실 없어도 돼 
다른 방식으로 프로젝트 개발을 진행한다면 말이야. 

odd-kit-template 을 보면 
루트 리드미와 docs 의 구현체라고 볼 수 있지. 가이드, 템플릿이라고 볼 수 있지. 

odd-kit-prompt.md 는 리드미와 어떻게 보면 비슷하다고 할 수 있어 
한장에 odd-kit 에 대한 모든 것을 설명해야해. 
철학과 역활 플로우 모두 다 말이야. 

docs/specs 에는 이 odd-kit-prompt.md 에 대한 스펙이 있어야할 수 있겠네 
근데 docs/specs 자체가 이미 그 스펙이라고 할 수 있어. 
사용자는 docs 를 설치하는게 아니고 odd-kit-template 을 설치하는 것일테니까 

odd-kit-template 의 폴더구조를 보면 
specs가 있네? 이건 docs/specs 에 있어야 할것 같은데? 
예상하건데 odd-kit-prompt 에 모든 내용을 쓰기가 어려우니까 분리를 해둔건가? 
근데 분리하면 참조가 잘 안되기도 하니까 일단 prompt 에 해당 스펙을 전부 기술하던가 해서 
이 폴더는 없애자 

그리고 나머지도 설명해야하는데 
내가 했던말 또 하려니까 지친다. 
이 세션에서 내가 한 말들도 필요한거 오리진 스펙에 넣어놔 
이 요청사항 자체도 오리진에 있어야 겠네.
