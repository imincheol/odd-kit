# 🏗️ Specifications (Specs)
> **Single Source of Truth for Requirements & Design**

이 디렉토리는 프로젝트의 **기획, 디자인, 설계** 문서를 보관하는 곳입니다.
코드를 작성하기 전, 반드시 이곳의 문서를 먼저 확인하고 정의된 스펙에 따라 구현하십시오.

## 📂 Structure

### 1_planning (기획)
- **What**: 사용자의 요구사항(User Story), 기능 정의서, 정책서
- **Goal**: "우리가 무엇을(What) 왜(Why) 만드는가?" 정의

### 2_design (디자인)
- **What**: UI/UX 시안, 와이어프레임, 디자인 시스템 토큰
- **Goal**: "어떤 모습(Look & Feel)으로 보여지는가?" 정의

### 3_markup (마크업)
- **What**: HTML 구조 설계, CSS 컴포넌트 설계
- **Goal**: "화면 구조(Structure)는 어떻게 잡을 것인가?" 정의

### 4_development (개발 설계)
- **What**: DB 스키마, API 명세, 시스템 아키텍처, 시퀀스 다이어그램
- **Goal**: "기술적으로(How) 어떻게 구현할 것인가?" 정의

---
**Tip**: 스펙이 변경되면 반드시 이곳의 문서를 먼저 업데이트하고, `history`에 남기십시오.
