# Origin: Project Structure & Philosophy

> **Date**: 2026-01-04

먼저 최상단 폴더와 최상단 파일들에 대해서 이야기해보자 

먼저 최상단 폴더에 대해서 말해야할거야 
- docs
- odd-kit-prompt-template
- .odd-kit

위 순서대로 중요도가 있다고 말할 수 있어.

docs 는 현재 프로젝트의 스펙이야. 
모든 내용이 docs 에 기록되어 있을거야. 
근데 현재 스펙에 가까워. 
과거에 있었던 스펙은 현재 스펙이 아니니까 지워져 있어 
현재 구현하는 현행 스펙을 말하는거야. 
현재를 표방하지 
모든 기능들, 현재의 기능들은 모두 스펙문서에 있어야해. 
스펙 문서에 없는 기능이라는건 없어. 

odd-kit-prompt-template
미래야.
일단 odd-kit-template 이 맞는 말이야. 
다른 프로젝트들이 odd-kit 을 설치할때 사용되는 틀에 가까워 
안에는 odd에 대한 동작에 대한 것들이 담겨 있어. 
미래에 이 프로젝트를 사용할 사람들이 쓰게 되는거지 

.odd-kit
docs 와 ok-template 폴더 자체가 이 프로젝트야. 
이 두개만 있어도 돼. 
그런데 이 프로젝트도 작업을 진행되어야할거야. 
저 두개가 현재와 미래라면 
.odd-kit 은 과거부터 현재까지야. 
지금까지 어떤 작업들이 진행되었는지가 기록되어 있고 
현재 어떤 스펙인지에 대해서 설명되어 있지 
odd-kit 프로젝트가 odd-kit 으로 작업이 되고 있다고 볼 수 있지 

위의 세가지 순서대로 진행되는게 기본이야. 
이건 오리진 스펙이지. 

그리고 그 이후에 리드미가 있어. 
리드미에는 위에 말한 내용을 추가로 표현되어 있어야해. 
리드미에서는 docs 와 odd-kit-template 두가지를 기준으로 작성되어 있지 
이 리드미를 보는 당신에게 프로젝트에 대해서 설명하고
현재 어떻게 만들어져 있으며 앞으로 당신은 어떻게 쓰면 된다 라는걸 알려주지 
프로젝트에 대한 이해가 쉽게 구조도가 비주얼하게 작성되어 있어 
설치 스크립트도 있지 
또한 세 폴더에 대한 설명도 있고 
주로 odd-template 이 어떻게 되어 있는지 철학과 개념, 플로우가 어떤지 
어떻게 사용하면 될지에 대해서 말하고 있을거야 
외부향 설명서이자 외부향 메뉴얼이야. 
영어로 한번 작성되어 있고 
그 후에 동일한 내용이 한글어로 중복으로 동일하게 작성되어 있어. 
영어는 세계인을 위해서 나타나는거고 
한국어는 나를 위해서 보여주는거야. 내가 보고 틀리면 내용을 수정해야하니까 
그래서 두개는 내용이 같아 .
이렇게 동일한 내용을 두번 보여주는건 프롬프트에 참조를 했을때도 좋아 
물론 리드미는 우리가 참조하지는 않지만 외부의 사용자들은 사용할 수도 있으니 
동일하게 두번 다른 언어로 반복하는것 자체도 스펙이라고 할 수 있지. 
이 리드미에 대한 스펙도 존재해야할거야.
